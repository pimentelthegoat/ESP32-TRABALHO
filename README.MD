# Trabalho: Velocimetro com ESP32 e Sensor Hall

Este projeto foi feito com a ideia de criar um sistema simples de monitoramento de velocidade usando um ESP32 e um sensor Hall KY-003.

A proposta e bem direta: o sensor Hall identifica a passagem de um ima, o ESP32 conta esses pulsos, calcula o RPM e uma estimativa de velocidade, e depois envia essas informacoes para um servidor local feito em Node.js. O servidor recebe os dados, salva no Supabase e deixa tudo disponivel para um painel web em tempo real.

O objetivo principal do trabalho nao foi criar um sistema industrial complexo, mas sim montar uma base organizada, facil de entender e que possa ser replicada por outros estudantes.

## Como o sistema funciona

O funcionamento geral ficou dividido em quatro partes:

1. O sensor Hall KY-003 detecta a passagem do ima.
2. O ESP32 conta os pulsos e calcula RPM e velocidade.
3. O backend local recebe os dados e salva no Supabase.
4. O frontend mostra os valores em um velocimetro digital atualizado em tempo real.

Na pratica, quando o eixo ou roda gira, o ima passa perto do sensor. Cada passagem gera um pulso. Com esses pulsos, o ESP32 consegue estimar quantas rotacoes estao acontecendo por minuto.

## Ligacao do sensor Hall KY-003

A ligacao usada foi simples:

```text
KY-003      ESP32
VCC    ->   3V3
GND    ->   GND
S      ->   GPIO 27
```

No codigo do ESP32, o pino usado para leitura do sensor e o `GPIO 27`. Se quiser usar outro pino, basta alterar essa parte diretamente no codigo aberto na Arduino IDE.

## Configuracao do Supabase

No Supabase, criei uma tabela chamada `leituras` para guardar os dados enviados pelo ESP32.

SQL usado:

```sql
create table leituras (
  id bigint generated by default as identity primary key,
  device_id text not null,
  rpm numeric not null,
  speed_kmh numeric not null,
  pulses integer not null,
  created_at timestamptz not null default now()
);
```

Essa tabela guarda:

- identificacao do dispositivo;
- RPM calculado;
- velocidade estimada em km/h;
- quantidade de pulsos;
- data e hora da leitura.

## Como rodar o backend

Primeiro entre na pasta do backend:

```powershell
cd backend
```

Instale as dependencias:

```powershell
npm.cmd install
```

Copie o arquivo de exemplo das variaveis de ambiente:

```powershell
copy .env.example .env
```

Depois abra o arquivo `.env` e coloque os dados do seu Supabase:

```env
SUPABASE_URL=https://SEU-PROJETO.supabase.co
SUPABASE_SERVICE_ROLE_KEY=SUA_CHAVE_SERVICE_ROLE
```

Para iniciar o backend:

```powershell
npm.cmd run dev
```

Se tudo estiver certo, deve aparecer algo parecido com:

```text
Backend local rodando em http://localhost:3000
WebSocket disponivel em ws://localhost:3000
```

## Como testar sem o ESP32

Antes de ligar o ESP32, da para testar o backend enviando uma leitura manual:

```powershell
curl -X POST http://localhost:3000/api/readings ^
  -H "Content-Type: application/json" ^
  -d "{\"device_id\":\"teste\",\"rpm\":860,\"speed_kmh\":10.5,\"pulses\":14}"
```

Depois acesse:

```text
http://localhost:3000/api/latest
```

Se aparecer a leitura enviada, o backend esta funcionando.

## Frontend na Vercel

O frontend esta dentro da pasta `frontend/`.

Para subir na Vercel, a ideia e criar um novo projeto e selecionar a pasta `frontend` como raiz do site. Como e HTML, CSS e JavaScript puro, nao precisa configurar comando de build.

Depois que o site estiver publicado, ele vai tentar conectar no backend local usando:

```text
http://localhost:3000
```

Isso funciona quando o site da Vercel e aberto no mesmo computador onde o backend esta rodando.

Se abrir o site pelo celular, por exemplo, `localhost` vai apontar para o proprio celular, nao para o computador. Nesse caso, seria necessario usar o IP do computador na rede ou algum tunel como ngrok.

## Codigo do ESP32 na Arduino IDE

O codigo do ESP32 nao fica mais dentro das pastas do projeto. Ele deve ser aberto ou colado direto na Arduino IDE.

Antes de enviar para a placa, e necessario alterar estas informacoes no codigo:

```cpp
const char* WIFI_SSID = "SEU_WIFI";
const char* WIFI_PASSWORD = "SUA_SENHA";
const char* SERVER_URL = "http://IP_DO_SEU_PC:3000/api/readings";
```

O `SERVER_URL` precisa usar o IP do computador onde o backend esta rodando. No Windows, esse IP pode ser encontrado com:

```powershell
ipconfig
```

Procure pelo endereco IPv4 da sua rede Wi-Fi.

## Ajustes importantes no calculo

No codigo usado na Arduino IDE existem alguns valores que podem ser ajustados:

```cpp
const int PULSES_PER_REVOLUTION = 1;
const float WHEEL_DIAMETER_METERS = 0.065;
```

O `PULSES_PER_REVOLUTION` indica quantos pulsos representam uma volta completa. Se for usado apenas um ima, o valor normalmente fica como `1`.

O `WHEEL_DIAMETER_METERS` representa o diametro da roda ou eixo usado no experimento. Esse valor influencia diretamente no calculo da velocidade em km/h.

## O que eu aprendi com o projeto

Com esse projeto foi possivel juntar conceitos de eletronica, programacao embarcada, backend e frontend em uma aplicacao so.

A parte mais interessante foi perceber como um simples pulso vindo de um sensor pode virar uma informacao visual em tempo real no navegador. Tambem deu para entender melhor como o ESP32 pode conversar com um servidor usando HTTP e como o WebSocket ajuda a atualizar a tela sem precisar recarregar a pagina.

Mesmo sendo um projeto simples, ele ja deixa uma base pronta para melhorias futuras, como historico de medicoes, graficos, autenticacao, mais sensores ou ate um painel com varios dispositivos.

## Conclusao

O projeto cumpriu a proposta de criar um velocimetro educativo usando ESP32 e sensor Hall KY-003. A estrutura ficou separada entre frontend, backend e codigo da placa usado diretamente na Arduino IDE, facilitando a manutencao e a explicacao do funcionamento.

A ideia principal foi mostrar, de forma pratica, como dados fisicos podem ser capturados por um sensor, processados por um microcontrolador, enviados para um servidor e exibidos em tempo real em uma interface web moderna.
